# `backend/core` 모듈 분석 우선순위 제안 보고서

이 문서는 `npr-gaussian-2d-prototype` 프로젝트의 백엔드 핵심 로직을 담당하는 `backend/core` 패키지의 모듈들을 효과적으로 분석하기 위한 우선순위를 제안합니다.

가장 효율적인 분석을 위해, 애플리케이션의 **가장 기본적인 데이터 단위(명사)부터 시작하여 → 데이터를 처리하는 핵심 로직(동사) → 부가적인 고급 기능 및 유틸리티 순서**로 진행하는 것을 추천합니다.

---

### 1순위: 핵심 데이터 구조 (Core Data Structures)

애플리케이션의 동작을 이해하기 위한 가장 기본적인 구성 요소들입니다.

1.  **`gaussian.py`**: **(최우선)** 프로젝트의 가장 작은 데이터 단위인 '가우시안 스플랫'을 정의합니다. 모든 렌더링과 계산은 이 데이터 구조를 기반으로 하므로 가장 먼저 분석해야 합니다.
2.  **`scene_data.py`**: 생성된 모든 가우시안 객체와 씬(Scene)의 전반적인 상태를 담는 '컨테이너'입니다. 데이터가 어떻게 집계되고 관리되는지 파악하는 데 필수적입니다.
3.  **`brush.py`**: 사용자의 입력을 통해 가우시안을 생성하는 '도구'인 브러시를 정의합니다. 사용자 상호작용과 직접적으로 연관된 핵심 데이터입니다.

### 2순위: 주요 데이터 처리 파이프라인 (Core Data Processing)

핵심 데이터들을 바탕으로 실질적인 '처리'를 수행하는 모듈들입니다.

4.  **`spline.py`**: 사용자의 마우스 입력을 부드러운 '경로(곡선)'로 변환하는 첫 번째 데이터 처리 단계입니다.
5.  **`renderer.py`**: `scene_data`에 담긴 가우시안들을 최종 **2D 이미지로 렌더링**하는 핵심 로직의 인터페이스 또는 기본 구현을 담당합니다. 애플리케이션의 최종 결과물이 어떻게 생성되는지 이해하는 데 가장 중요합니다.
6.  **`renderer_gpu.py` / `renderer_gsplat.py`**: `renderer.py`의 구체적인 구현체들로, GPU를 사용한 고성능 렌더링의 상세 로직을 담고 있습니다.

### 3순위: 고급 기능 및 서브 시스템 (Advanced Features & Sub-systems)

핵심 파이프라인에 부가 가치를 더하는 독립적인 기능 모듈들입니다.

7.  **`deformation.py` / `deformation_gpu.py`**: 생성된 스플라인 곡선을 따라 브러시를 자연스럽게 휘게 만드는 고급 시각 효과 기능입니다.
8.  **`brush_manager.py`**: 브러시 라이브러리 전체를 관리하고, 브러시 생성/삭제 등의 UI 상호작용과 관련된 로직을 처리합니다.
9.  **`brush_converter.py` / `depth_estimator.py`**: 사용자가 업로드한 2D 이미지를 새로운 3D 브러시로 변환하는, 독립적인 서브 시스템입니다.
10. **`inpainting.py`**: 붓질이 겹쳤을 때 경계를 부드럽게 처리하는 고급 시각 효과 기능입니다.

### 4순위: 유틸리티 및 보조 모듈 (Utilities & Helpers)

위의 핵심 로직들을 보조하는 도구 모음입니다.

11. **`quaternion_utils.py`**: 3D 회전 계산에 사용되는 수학 함수 모음입니다.
12. **`optimizer_torch.py`**: 특정 상황에서 PyTorch를 사용해 씬을 최적화하는 데 사용될 수 있는 특수 목적 모듈입니다.
13. **`debug_visualizer.py`**: 디버깅 목적으로 내부 데이터를 시각화하는 코드로, 핵심 기능보다는 개발 편의성을 위한 모듈입니다.
